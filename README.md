# permreg — 利用者権限登録リスト 管理 bookmarklet

システム利用者の権限登録用 SharePoint リストを管理するための bookmarklet 群。
サーバー不要・インストール不要で、SharePoint サイトのページ上から REST API で操作する。

## フェーズ1(現在): 組織区分マスタ管理

「組織区分 第1階層 / 第2階層」のマスタを管理する UI。
第1階層ごとに表示する第2階層が異なるため、第2階層は第1階層への参照(Lookup)を持つ。

起動して「初期セットアップ」を押すと、対象サイトに次の2リストを作成する:

| リスト | 列(内部名) | 種類 |
|---|---|---|
| 組織区分第1階層マスタ | Title(名称) / SortOrder(並び順) / Active(有効) | テキスト / 数値 / はい・いいえ |
| 組織区分第2階層マスタ | Title(名称) / Level1(第1階層) / SortOrder / Active | テキスト / **Lookup→第1階層** / 数値 / はい・いいえ |

管理UIでできること: 追加 / 名称変更 / 削除 / ▲▼で並べ替え / 有効・無効の切替。
第1階層を選択すると、その配下の第2階層だけが右ペインに表示される。

## ビルドとインストール

```sh
npm install
npm run build
```

- `dist/install.html` をブラウザで開き、ボタンをブックマークバーへドラッグ
- 対象の SharePoint サイトを開いた状態でブックマークをクリックして起動

## 動作確認(モック)

SharePoint なしで UI を確認するハーネス:

```sh
npm run build
# test/harness.html をブラウザで開く(REST はメモリ上のモックに置き換わる)
```

## 必要な権限

- マスタリストの作成(初期セットアップ)にはサイトの「フルコントロール」または「デザイン」権限が必要
- 作成後のマスタ編集は対象リストへの「投稿」権限があればよい

## ロードマップ

- フェーズ2: 利用者一覧リスト本体(利用者名 / 会社名 / メールアドレス / 変更区分 / 権限 / 組織区分第1・第2階層 / 特記事項 / システム反映日 / システム削除)の作成と、マスタ連動の入力フォーム UI
- フェーズ3: 第1階層の選択に応じて第2階層の選択肢を絞り込む登録フォーム、一括取込など
