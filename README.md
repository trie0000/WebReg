# permreg — 利用者権限登録リスト 管理 bookmarklet

システム利用者の権限登録用 SharePoint リストを管理するための bookmarklet 群。
サーバー不要・インストール不要で、SharePoint サイトのページ上から REST API で操作する。

## フェーズ1(現在): 組織区分マスタ管理

「組織区分 第1階層 / 第2階層」のマスタを管理する UI。
第1階層ごとに表示する第2階層が異なるため、第2階層は第1階層への参照(Lookup)を持つ。

起動して「初期セットアップ」を押すと、対象サイトに次の2リストを作成する:

| リスト | 列(内部名) | 種類 |
|---|---|---|
| 組織区分第1階層マスタ | Title(名称) / SortOrder(並び順) / Active(有効) | テキスト / 数値 / はい・いいえ |
| 組織区分第2階層マスタ | Title(名称) / Level1(第1階層) / SortOrder / Active | テキスト / **Lookup→第1階層** / 数値 / はい・いいえ |

管理UIでできること: 追加 / 名称変更 / 削除 / ▲▼で並べ替え / 有効・無効の切替。
第1階層を選択すると、その配下の第2階層だけが右ペインに表示される。

## ビルドとインストール(ローダ方式)

ビルドに必要なのは **Python 3.8+ のみ**(標準ライブラリだけ。npm / Node / pip 不要):

```sh
python3 build.py    # Windows: python build.py
```

src/ を依存順に連結して1つの IIFE にまとめ、軽量 minify(コメント・インデント・空行の除去。
識別子の短縮はしない)を掛けて dist/ 一式を生成する。esbuild の minify と比べて +5% 程度のサイズ差。

1. `dist/` 一式を SharePoint の **ドキュメント/permreg/** にアップロード
2. `dist/install-loader.html` をブラウザで開き、ボタンをブックマークバーへドラッグ
3. 対象の SharePoint サイトを開いた状態でブックマークをクリックして起動

bookmarklet はローダ(約3KB)のみ。本体(`permreg.bundle.js`)は起動のたびに
`version.txt` を確認して SP から最新を読み込むため、**dist を SP に再アップするだけで全員に反映**される。
オフライン/SP 未配置で使う場合のみ `dist/install.html`(丸ごと埋め込み版・自動更新なし)。

### 起動中の自動更新

起動中も読込元(SP またはローカル)の `version.txt` を 30秒ごと+タブ復帰時に確認し、
新しい版があれば**更新モーダル**を表示 → 「今すぐ更新」でその場で新 bundle を再読込する
(ホストページのリロード・ブックマーク再クリック不要。中継サーバも不要)。
「後で」を選んだ版は再通知しない(さらに新しい版が出たら通知)。
版比較は安定識別子 `<ver>-<srcSha8>` の一致判定。

## 開発者モード(ローカルフォルダ参照)

ビルドした dist/ をローカルから配信して、SP へのアップロードなしで動作確認できる:

```sh
python3 dev/serve.py   # http://127.0.0.1:18086/permreg/ → dist/ (CORS付き・全リクエストをログ)
```

アプリの「設定」メニュー → 配信元を「ローカル開発サーバ(開発者モード)」にして保存 →
次回のブックマークレット起動からローカルの dist/ が読み込まれる。
URL は設定画面で変更可能(既定 `http://127.0.0.1:18086/permreg`)。読込失敗時は alert で通知される。

**配信フォルダ**も設定画面から変更できる(`permreg.bundle.js` を含むフォルダの絶対パスを指定して保存)。
サーバ側 API は `GET/POST /permreg/bundle-dir`(POST は存在チェックあり)。再起動で既定
(`PERMREG_BUNDLE_DIR` または dist/)に戻る。別ブランチのビルド先や持ち込んだ dist を即配信する用途。

## 必要な権限

- マスタリストの作成(初期セットアップ)にはサイトの「フルコントロール」または「デザイン」権限が必要
- 作成後のマスタ編集は対象リストへの「投稿」権限があればよい

## マスタの「リストへ反映」

マスタ管理画面の「リストへ反映」ボタンで、組織区分マスタを **利用者一覧リスト**の列構成に反映する:

| 対象 | 形式 |
|---|---|
| 組織区分第1階層 | 選択肢(Choice)列 `OrgLevel1`。選択肢 = 有効な第1階層マスタ |
| 第2階層(各マスタ) | はい/いいえ列(内部名 `L2_<マスタID>` — 改名しても安定)。同名は「名称(第1階層名)」で一意化 |
| 組織区分第2階層 | 集計列 `OrgLevel2`。「☑欧州/◽北米/…」形式で表示(式は反映のたびに再生成) |

- リストが無ければ基本列(利用者名/会社名/メールアドレス/変更区分/権限/特記事項/システム反映日/システム削除)ごと作成
- 冪等。マスタの**改名は列の表示名に追従**。**無効/削除では列を消さない**(既存行のデータ保全。選択肢・集計式から外れるだけ)
- 親の第1階層が無効なら、その配下の第2階層も反映対象外
